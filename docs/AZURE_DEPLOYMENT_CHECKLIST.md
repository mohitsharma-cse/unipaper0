# Azure Deployment Checklist

This project is a full-stack Node/Express app that also serves the static frontend. Deploy it to Azure App Service, not Azure Static Web Apps, unless the backend is split into a separate API service.

## Microsoft/Azure Requirements

- App Service runtime: Node.js 22, matching `package.json`.
- Startup command: `npm start`.
- Public health check: `/api/ready`.
- Build verification: `npm run build`.
- Required files in the deploy package: `package.json`, `package-lock.json`, `backend/src/**`, root frontend assets, `node_modules/**`, and `web.config` for Windows App Service.
- Do not deploy `.env`; put secrets in Azure App Service Configuration.
- Keep MongoDB Atlas network access, storage tokens, and JWT secret outside the repository.

## Azure App Settings

Set these in Azure Portal > App Service > Environment variables:

```env
NODE_ENV=production
WEBSITE_NODE_DEFAULT_VERSION=~22
MONGODB_URI=mongodb+srv://...
JWT_SECRET=use_a_long_random_secret_at_least_32_chars
JWT_EXPIRES_IN=24h
CLIENT_URL=https://your-app.azurewebsites.net
API_PUBLIC_URL=https://your-app.azurewebsites.net
COOKIE_SAME_SITE=lax
COOKIE_SECURE=true
STORAGE_PROVIDER=uploadthing
UPLOADTHING_TOKEN=...
UPLOADTHING_APP_ID=...
UPLOADTHING_STORAGE_KEY=uploadthing-1
UPLOADTHING_STORAGE_LABEL=UploadThing 1
```

For a separate frontend domain, set `CLIENT_URL` to both domains, set `COOKIE_SAME_SITE=none`, and keep `COOKIE_SECURE=true`.

## Post-Deploy Test Flow

1. Open `/api/health`; it should return `status: ok`.
2. Open `/api/ready`; it should return `ready` after MongoDB connects.
3. Open `/admin`, log in, and confirm the auth cookie is set.
4. Create a folder, upload a PDF, preview it, download it, then delete it.
5. Open the public site, search, browse folders, submit a review, and approve it from admin.
